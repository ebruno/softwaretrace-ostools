3 ## Background Information ##
4 When attempting to deal with process management issues developers often take the 'quick approach' and simply use the system command to shell out to command line tools or custom stripts to obtain this type of information.
5 
6 The Software Engineering Institute (SEI) at Carnegie Mellon University is part of the Computer Emergency Response Team (CERT)
7 SEI CERT publishes coding standards for various languages (C, C++, Java and Perl).
8 The C specification can found here:[SEI CERT C Standard](https://wiki.sei.cmu.edu/confluence/display/c/SEI+CERT+C+Coding+Standard)
9 The recommendation we are interested in is [ENV33-C. Do not call system](https://wiki.sei.cmu.edu/confluence/pages/viewpage.action?pageId=87152177)
10 
11 In brief what ENV33-C states is shown below:
12 
13 > The C Standard system() function executes a specified command by invoking an implementation-defined command processor,
14 > such as a UNIX shell or CMD.EXE in Microsoft Windows. The POSIX popen() and Windows _popen() functions also invoke a command
15 > processor but create a pipe between the calling program and the executed command, returning a pointer to a stream that can
16 > be used to either read from or write to the pipe [IEEE Std 1003.1:2013].
17 >
18 > Use of the system() function can result in exploitable vulnerabilities, in the worst case allowing execution of arbitrary system commands. Situations in which calls to system() have high risk include the following:
19 >
20 > * When passing an unsanitized or improperly sanitized command string originating from a tainted source
21 >
22 > * If a command is specified without a path name and the command processor path name resolution mechanism is accessible to an attacker
23 >
24 > * If a relative path to an executable is specified and control over the current working directory is accessible to an attacker
25 >
26 > * If the specified executable program can be spoofed by an attacker
27 >
28 > Do not invoke a command processor via system() or equivalent functions to execute a command.
29 
30 The recommendation ENV33-C is actually applicable to any programming language in an Linux/Unix platform that has a "system" command feature.
31 
32 The functions in this library allow a developer to obtain the same type of information when API's do not exist to obtain the information.
